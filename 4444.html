Краткий конспект по Git Hub/Git Git - программа Git Hub - это облачное хранилище, где можно публиковать свой код и все изменения в нем, внесенные с помощью Git / Vs Code ВНИМАНИЕ, пока выполняются процедуры по заливке файлов на Git Hub, в открытых файлах и папках лучше не производить изменения и ничего в них не писать (до шага 15). 1) Git Hub нужен для того, чтобы работать вместе с командой над проектом, а также чтобы добавлять новые версии кода, сохраняя при этом старые. При этом залитые на Git Hub файлы с кодом могут быть открытыми для публичного просмотра или приватными. 2) Git = win+R —> cmd на Windows. Подавать команды можно через терминал на VS Code, через командную строку win+R или через Git Bash, который идет в пакете, когда мы скачиваем Git Hub на компьютер. 3) Чтобы начать работу с Git Hub на компьютере создаем папку. Далее заходим в VS Code и там открываем эту папку и создаем файл. Этот файл уже можно отправить в совместный доступ на Git Hub. Сделать это можно либо через кнопку на левой вертикальной панели в VS Code, если нажать на изображение вилки (разветвленный проводок) и далее нажать на кнопку Initialize Repository. Нам покажется самая первая версия кода. 4) ЛИБО можно в VS Code вызвать командную строку, нажав на горизонтальной верхней панели Terminal —>New Terminal и указать команду git init (по умолчанию программа откроет уже нужный файл на терминале) 5) Чтобы удалить ненужный репозиторий на этом шаге, можно зайти с компьютера в папку, где сохранен код, и там нажать Вид → Показать скрытые папки → Удалить высветившуюся папку. Таким действием мы сотрем команду git init и все, что терминал делал с нашим файлом до этого. 6) Заходим в Git Hub и нажимаем “создать новый репозиторий”(create)По умолчанию репозиторий указан публичный. В репозитории будет список необходимых команд, которые нужно сделать, чтобы загрузить файлы в облако. 7) Далее пишем в терминале в VS Code команду git add . (именно в таком виде - после точки идет пробел). Данная команда добавит все имеющиеся в папке файлы прямиком в репозиторий. Если же надо добавить не все, а выборочные файлы, то вместо точки нужно написать git add имя файла в терминале. 8) далее вводим команду git commit -m "first commit" Буква -m означает, что после нее будет послание, месседж. Нам Git Hub предлагает написать “first commit”. Если все прошло хорошо, то в левой части VS Code появится кнопка Publish Branch (Опубликовать Branch) 9) Данный шаг обязателен только для тех, кто впервые создал аккаунт. Git Hub в терминале запросит выполнить серию команд: git config --global user.email "адрес эл.почты" git config --global user.name “имя пользователя в Git Hub" Они выполняются всего один раз. Если терминал запросил такое, то после всех действий с почтой и именем команду git commit -m "first commit" нужно повторить. И проверить, появилась ли кнопка Publish Branch. 10) Проверить, что все ок, можно двумя путями а) через компьютер / локальный диск С→ Пользователи → файл .gitconfig б) ввести в терминале команду git config –list (перед list два дефиса). Чтобы выйти из режима просмотра, нажимаем клавишу q 11) Данный шаг не является обязательным. Но, если при установке Git основной папкой была назначена папка Master, то придется переменить ее на папку Main. Быстро проверить, какая папка была назначена, можно, посмотрев в левый нижний угол терминала в VS Code. Итак, если там указано Master, то нужно выполнить команду git branch -M main т.е. переключиться на Main 12) Далее надо перейти на сайт Git Hub и скопировать из списка рекомендуемых команд команду + ссылку на репозиторий. У меня в примере это git remote add origin https://github.com/Keith914/test_fruit.git 13) Последняя команда из списка: git push -u origin main что означает, что мы хотим передать скопированные данные с ссылки https://github.com/Keith914/test_fruit.git на ветку main. U в этом случае означает url 14) Далее появится окно авторизации от Git. Надо нажать верхнюю кнопку Sign up with your browser А далее нажать зеленую кнопку внизу - Authorize git-ecosystem Если сейчас обновить страницу на Git Hub, то мы увидим, что файлы записались в репозиторий и можно посмотреть их код. 15) А вот теперь можно менять/добавлять код в файлах в VS Code. Когда изменения добавлены, снова пишем команды git add . , далее git commit -m "имя коммита" и далее git push -u origin main Если зайти в залитый файл на Git Hub, то в правом верхнем углу в Git Hub есть History - там можно посмотреть историю всех коммитов в файл. До этого мы, в основном, рассматривали действия на терминале. Но есть и альтернативный путь: через систему управления версиями (иконка на левой панели столбцом) в VS Code (см тут: https://postimg.cc/ykDHh7Jh) Рассмотрим его, чтобы понимать связь команд в терминале и в VS Code. 1) Создадим новую папку и в ней - новый файл в VS Code. Просто создать файл - недостаточно. Нужно, чтобы он именно был в папке. 2) Нажмем на панели VS Code (далее - просто “панель”) кнопку с надписью “Инициализировать репозиторий” / Inintialize Repository (если у нас нет русскоязычного плагина на VS Code) 3) Далее у нас появится окошко, в котором мы можем вписать название коммита: https://postimg.cc/7fJSFTG5 Пусть названием будет first commit (без кавычек), т к это первый коммит для этого файла. И далее нажимаем кнопку “Фиксация” либо Commmit в англоязычной версии. Далее программа нас спросит, хотим ли мы подготовить изменения и зафиксировать их напрямую. Мы нажимаем “Да”. После чего кнопка “Фиксация” заменяется кнопкой “Опубликовать Branch”. Нажатие кнопки “Фиксация” заменяет сразу две команды в терминале 1)git add . 2) git commit -m “название коммита” 4) Нажимаем эту кнопку. Нажимаем “Разрешить” на появившемся алерте. 5) Нас перекидывает на Git Hub в браузере. Там жмем Continue → Authorize VS Code → Открыть приложение 6) VS Code открывается и в строке поиска нам предлагается выбрать тип репозитория: публичный или приватный. Выбираем. https://postimg.cc/NKBGtWJW 7) Готово. Проверяем на Git Hub - все работает. 8) Если внесли изменения в код на VS Code, то повторяем действие 3) Еще одной альтернативой является комбинированный способ отправки данных на Git Hub. А именно: использовать боковую панель VS Code и панель графического интерфейса в VS Code (это три точечки рядом с названием репозитория). Пошаговое выполнение: 1) Открыть папку, создать файл, нажать кнопку “Инициализировать репозиторий” на боковой панели с ветвью в VS Code и далее написать название коммита и нажать кнопку “Фиксация”. Этими действиями мы выполнили три первых команды из обязательного списка, который требуется на Git Hub. 2) Данный шаг для справки. Если надо переименовать ветку в графическом интерфейсе VS Code, то выбрать на верхней панели в меню три точки рядом с названием Репозитория → Ветвь → Переименовать Ветвь (аналог шага 11 при работе с терминалом). Вот, как это выглядит: https://postimg.cc/PC8tmY3Q п.с. сейчас по дефолту должно быть main. В более ранних версиях Git Hub было master, поэтому система может все еще относить репозитории туда. 3) Переходим на Git Hub и там создаем новый репозиторий. Возвращаемся в VS Code. 4) Чтобы выполнить команду git remote add origin название ссылки мы копируем ее с Git Hub и вставляем в VS Code через графический интерфейс “Удаленный” → Добавить удаленный репозиторий (remote → add remote) → в черной адресной строке выбираем “Добавить репозиторий из Git Hub” https://postimg.cc/PLT5TNyb В адресной строке указываем скопированную ссылку с Git Hub и хатем вводим название 5) Чтобы выполнить команду git push -u origin main, опять нажимаем три точки в VS Code и выбираем из списка “Вытягивание / отправка” (pull / push) → “Отправить в” (push to) https://postimg.cc/F1jCJRVk → выбрать, куда отправить (название репозитория будет в подсказке-ссылке) 6) После этого можно вносить изменения в файлы и для команды add . использовать кнопку “Фиксация” + указать название коммита на боковой панели в VS Code 7) Далее перейти в три точки графического интерфейса VS Code и выбрать из списка “Вытягивание / отправка” (pull / push) → “Отправить в” (push to) → выбрать, куда отправить (название репозитория будет в подсказке-ссылке) РЕЗЮМЕ: основные команды, которые нужно выполнить при работе с Git Hub: git init git add . git commit -m "first commit" git remote add origin https://github.com/Keith914/название.git git push -u origin main Выполнить эти команды можно тремы способами: через терминал через VS Code - боковая панель слева (значок ветви) комбинированным способом: через боковую панель слева (значок ветви) + панель графического интерфейса (три точечки на панели вверху, рядом с названием репозитория в VS Code) После того, как проект создан и размещен на Git Hub, в него можно вносить изменения и снова коммитить. Как клонировать репозиторий (т.е. скачать себе на комп все файлы из папки )? 1. Зайти в репозиторий 2. Начать зеленую кнопку Code в правом верхнем углу. Альтернативный вариант клонирования: Правой кнопкой мыши на рабочем столе компьютера выбрать Git Bash Скопировать ссылку с Git Hub под все той же зеленой кнопкой Вернуться в Git Bash и задать команду git clone /вставить скопированную ссылку/. Далее нажать ввод. Все скопируется в отдельную папку с тем же названием, что и на Git Hub, на рабочий стол. Зайти в скопировавшуюся папку и нажать “показывать скрытые файлы”. В появившейся скрытой папке можно будет увидеть всю информацию. РАСШИРЕНИЕ GIT GRAPH Данное расширение устанавливается в VS Code через поиск расширений. Автор mhutchie Расширение позволяет отследить, не выходя из VS Code, все изменения в файле, включая имя создателя и его эл. почту. После установки нужно зайти в раздел, где мы коммитим изменения (иконка веток) и там будет иконка этого расширения на горизонтальном меню в виде стрелок, указывающих вверх, что повторяет цветную иконку расширения при установке. Если нажать на иконку на панели меню, то откроется graph-файл со всей подробной информаицей. Это очень удобно в том плане, что а) не надо лазить в Git Hub, чтобы посмотреть историю изменения б) все наглаядно в) возможны локальные изменения именно в открытой программе VS Code, которые не сохраняются в Git Hub
