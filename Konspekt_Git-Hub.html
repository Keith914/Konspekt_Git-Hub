Краткий конспект по Git Hub/Git Взято отсюда: https://www.youtube.com/watch?v=zUzQG7hmdNM канал ITDoctor Git - программа Git Hub - это облачное хранилище, где можно публиковать свой код и все изменения в нем, внесенные с помощью Git / Vs Code ВНИМАНИЕ, пока выполняются процедуры по заливке файлов на Git Hub, в открытых файлах и папках лучше не производить изменения и ничего в них не писать (до шага 15). 1) Git Hub нужен для того, чтобы работать вместе с командой над проектом, а также чтобы добавлять новые версии кода, сохраняя при этом старые. При этом залитые на Git Hub файлы с кодом могут быть открытыми для публичного просмотра или приватными. 2) Git = win+R —> cmd на Windows. Подавать команды можно через терминал на VS Code, через командную строку win+R или через Git Bash, который идет в пакете, когда мы скачиваем Git Hub на компьютер. 3) Чтобы начать работу с Git Hub на компьютере создаем папку. Далее заходим в VS Code и там открываем эту папку и создаем файл. Этот файл уже можно отправить в совместный доступ на Git Hub. Сделать это можно либо через кнопку на левой вертикальной панели в VS Code, если нажать на изображение вилки (разветвленный проводок) и далее нажать на кнопку Initialize Repository. Нам покажется самая первая версия кода. 4) ЛИБО можно в VS Code вызвать командную строку, нажав на горизонтальной верхней панели Terminal —>New Terminal и указать команду git init (по умолчанию программа откроет уже нужный файл на терминале) 5) Чтобы удалить ненужный репозиторий на этом шаге, можно зайти с компьютера в папку, где сохранен код, и там нажать Вид → Показать скрытые папки → Удалить высветившуюся папку. Таким действием мы сотрем команду git init и все, что терминал делал с нашим файлом до этого. 6) Заходим в Git Hub и нажимаем “создать новый репозиторий”(create)По умолчанию репозиторий указан публичный. В появившейся скрытой папке можно будет увидеть всю информацию. РАСШИРЕНИЕ GIT GRAPH Данное расширение устанавливается в VS Code через поиск расширений. Автор mhutchie Расширение позволяет отследить, не выходя из VS Code, все изменения в файле, включая имя создателя и его эл. почту. После установки нужно зайти в раздел, где мы коммитим изменения (иконка веток) и там будет иконка этого расширения на горизонтальном меню в виде стрелок, указывающих вверх, что повторяет цветную иконку расширения при установке. Если нажать на иконку на панели меню, то откроется graph-файл со всей подробной информаицей. Это очень удобно в том плане, что а) не надо лазить в Git Hub, чтобы посмотреть историю изменения б) все наглаядно в) возможны локальные изменения именно в открытой программе VS Code, которые не сохраняются в Git Hub Интересные команды и АЛИАСЫ (сокращенные команды git status –short (команда гит статус пробел два дефиса шорт) - тут указана краткая информация о файле. git status - указана полная информация о файле git log --graph --decorate --all - еще более полная информация о коммитах (такая же, как и при использовании кнопки расширения Git Graph) git branch - показывает, на какой ветке мы находимся git branch -M main (или название другой ветки) - переключение между ветками (где main - это название ветки). И также команда git branch может создавать ветки. checkout - переключение между ветками, например git checkout -b название ветки - создать новую ветку и сразу переключиться на неё Команды в GIT HUB бывают очень длинные, но их можно сокращать. И вот для этого и нужны алиасы (сокращения). Давайте откроем файл .gitconfig в папке Диск С → Пользователи → Админ → .gitconfig Этот файл относится к категории скрытых файлов, поэтому, чтобы его увидеть в разделе ВИД надо поставить галочку в чек-боксе “Скрытые элементы” (эти инструкции актуальны для ОС WINDOWS) Файл .gitconfig можно открывать в блокноте или другом текстовом редакторе и там вносить изменения. Мы же откроем его в VS Code. Чтобы создать алиас напишем следующую команду: [alias] s = status --short st = status l = log --oneline --graph --decorate --all g = log --graph --abbrev-commit --decorate --all --format=format:'%C(bold blue)%h%C(reset) - %C(bold cyan)%aD%C(dim white) - %an%C(reset) %C(bold green)(%ar)%C(reset)%C(bold yellow)%d%C(reset)%n %C(white)%s%C(reset)' br = branch co = checkout Эти алиасы создал автор канала. Можно и нужно создать свои тоже! Теперь можно написать короткое название команды и GIT ее распознает и выполнит. Таким образом мы зафиксировали промежуточные изменения, которых еще нет в Git Hub, но они готовы к отправке. (это также могут назвать стэйдж). МЯГКИЙ СБРОС, то тогда выполним в терминале команду soft: git reset --soft HEAD^1 Нас отбросит на стадию стэйджа, но при этом ничего не сотрется в самом файле в VS Code Альтернатива отмены коммита с помощью графического интерфейса (через меню): для мягкого сброса: в меню CHANGES выбрать Зафиксировать → Отменить последнюю фиксацию Вот так: https://postimg.cc/zHC1jVGz Также мы можем свободно переключаться между версиями нашего файла в разделе GIT GRAPH, нажав на ту версию, из которой хотим работать: https://postimg.cc/8JVFPf9D Но есть один прикол: если через GIT GRAPH возвращаться назад и делать изменения в предыдущих коммитах, то эти изменения не пойдут дальше стэйджа, видимо, во избежание конфликта, и синхронизацию с Git Hub система не сделает. Выберите репозиторий, который хотите клонировать, и укажите локальную папку для сохранения. ЛИБО Через командную строку (Git CLI): Установите Git на новый компьютер. Откройте командную строку (терминал). Перейдите в папку, в которой хотите сохранить репозиторий. Выполните команду: git clone (URL репозитория можно найти на странице GitHub в разделе "Code"). 2. Открытие файла: После клонирования репозитория перейдите в папку, в которой он сохранен. Найдите нужный файл в этой папке. Щелкните по нему правой кнопкой мыши и выберите "Открыть с помощью..." (или аналогичную опцию, в зависимости от операционной системы). Выберите программу, в которой хотите открыть файл (например, текстовый редактор, IDE и т.д.).
